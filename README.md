# Community Feedback App

אתר משוב לאונסייט קומיוניטי. כל שיעור מקבל QR ייעודי, המשתתפים סורקים וממלאים סקר קצר, ובסוף האונסייט יש סקר סיכום מקיף. דף אדמין מוגן ב-PIN מציג ניתוח חי ומאפשר ייצוא ל-CSV.

**Stack:** React + Vite, Supabase (Postgres) לאחסון, Netlify לדפלוי.

---

## 🚀 הוראות התקנה צעד-צעד

### שלב 1 — Supabase

1. היכנסי ל-[supabase.com/dashboard](https://supabase.com/dashboard) ופתחי את הפרויקט שלך (או צרי חדש).
2. בתפריט הצדדי לחצי **SQL Editor** → **New Query**.
3. הדביקי את כל התוכן של `supabase_schema.sql` ולחצי **Run**. זה יוצר 2 טבלאות עם RLS פתוח לקריאה/כתיבה אנונימית.
4. בתפריט הצדדי לחצי **Settings** ⚙️ → **API**. רשמי לעצמך:
   - **Project URL** (משהו כמו `https://abcdef.supabase.co`)
   - **anon public** key (מתחיל ב-`eyJ...`)

### שלב 2 — להריץ מקומית (אופציונלי, לבדיקה)

```bash
npm install
cp .env.example .env
# ערכי את .env והדביקי את ה-URL וה-anon key מסעיף 1.4
npm run dev
```

ייפתח לוקלי על `http://localhost:5173`. בדקי:
- דף הבית — בחירת שיעור והצגת QR
- `#l/d1-s1` — דף משוב לשיעור
- `#admin` — מסך אדמין (PIN ברירת מחדל: `7777`)

### שלב 3 — דפלוי ל-Netlify

**אפשרות א — דפלוי בגרירה (הכי מהיר, ללא git):**

1. הריצי `npm install && npm run build`. זה יוצר תיקיית `dist/`.
2. היכנסי ל-[app.netlify.com](https://app.netlify.com) → **Add new site** → **Deploy manually**.
3. גררי את התיקייה `dist` לחלון.
4. אחרי הדפלוי, לחצי **Site configuration** → **Environment variables** → **Add variable**, והוסיפי:
   - `VITE_SUPABASE_URL` = ה-URL מסעיף 1.4
   - `VITE_SUPABASE_ANON_KEY` = ה-anon key מסעיף 1.4
   - `VITE_ADMIN_PIN` = קוד PIN שתבחרי (אופציונלי, ברירת מחדל 7777)
5. **חשוב** — אחרי הוספת ה-env vars, את חייבת לעשות rebuild. הריצי שוב `npm run build` לוקלי וגררי את `dist` החדש.

**אפשרות ב — דפלוי מ-Git (מומלץ, מתעדכן אוטומטית):**

1. דחפי את הפרויקט ל-GitHub/GitLab.
2. ב-Netlify → **Add new site** → **Import existing project** → בחרי את הריפו.
3. Build command: `npm run build`, Publish directory: `dist` (אמור להיות אוטומטי כי יש `netlify.toml`).
4. **לפני שמסיימים** — הוסיפי את 3 ה-Environment Variables (כמו באופציה א, סעיף 4).
5. **Deploy site**.

### שלב 4 — לבדוק שהכל עובד

1. פתחי את ה-URL של Netlify (משהו כמו `https://your-site.netlify.app`).
2. בחרי שיעור, סרקי את ה-QR מהטלפון. את אמורה לראות את דף המשוב.
3. מלאי משוב לדוגמה.
4. פתחי `your-site.netlify.app/#admin`, הזיני PIN, ראי שהמשוב הופיע.

## 🎤 איך להשתמש ביום האונסייט

1. **לפני שמתחילים:** פתחי את האתר על המסך הגדול. יש שני מסכים שתעבדי איתם:
   - דף הבית — להצגת QR בסוף כל שיעור.
   - `#admin` — לפתוח על מסך אישי / מחשב נפרד אם תרצי לעקוב בזמן אמת.

2. **בסוף כל שיעור:** בדף הבית, בחרי את השיעור הרלוונטי מהרשימה. ה-QR מתחלף אוטומטית. אמרי לקהל "סורקים וממלאים, לוקח דקה" והציגי על המסך 60–90 שניות.

3. **בסוף האונסייט:** בחרי את ה-QR השני (סקר סיכום) והציגי אותו לזמן ארוך יותר — 3–5 דקות.

4. **אחרי האונסייט:** היכנסי ל-`#admin`, סקרי את הנתונים, הורידי CSV לאקסל אם תרצי לחתוך נתונים נוספים.

## 🔐 על אבטחה ופרטיות

- הסקר אנונימי: לא נשמרים שמות, אימיילים, או מזהים אישיים.
- Supabase anon key הוא מפתח public — בטוח להציג אותו בקוד הלקוח.
- ה-PIN של מסך אדמין הוא הגנה רכה (UX) ולא אבטחה אמיתית. **אם משוב שיעור הוא רגיש**, אפשר להחליף את מדיניות ה-RLS של `select` לכזו שדורשת `authenticated`, ולעטוף את מסך אדמין ב-Supabase Auth. כרגע הקוד פתוח לקריאה לכל מי שיודע את ה-URL של ה-Supabase project.

## 📝 איך להוסיף/לשנות שיעורים

ערכי את המערך `LECTURES` בראש הקובץ `src/App.jsx`. כל שורה היא:

```js
{ id: 'd1-s1', day: 1, title: '...', speaker: '...', type: 'Talk' | 'Workshop' }
```

ה-`id` הוא מה שמופיע בקישור (`#l/d1-s1`), אז אם משנים `id` של שיעור שכבר התחילו לאסוף עליו משוב, היסטוריית הניתוח תאבד את הקישור.

## 📝 איך לשנות את שאלות הסקר

- **משוב קצר**: שני את המערכים `EMOJI_OPTIONS` ו-`TAKEAWAY_OPTIONS` בראש `App.jsx`.
- **סקר סיכום**: שני את `IMPACT_AREAS`, או ערכי ישירות בקומפוננטה `FinalPage`.
- **שדות פרופיל**: שני את `ROLES` ו-`SENIORITY`.
